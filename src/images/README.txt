IMAGES FOLDER - PLACEHOLDER FILES
==================================

This folder contains placeholder images that should be replaced with your actual project screenshots.

REQUIRED REPLACEMENTS:
----------------------

1. desktop.png
   - Purpose: Desktop/landscape project screenshots
   - Recommended size: 1920x1080px or 1200x800px
   - Format: PNG or JPG (PNG preferred for screenshots)
   - Max file size: 200KB (compress with TinyPNG or ImageOptim)
   - Currently: Gray placeholder (1920x1080, ~50KB)

2. mobile.png
   - Purpose: Mobile app screenshots  
   - Recommended size: 375x667px (iPhone) or 360x640px (Android)
   - Format: PNG or JPG
   - Max file size: 150KB
   - Currently: Gray placeholder (375x667, ~30KB)

HOW TO REPLACE:
---------------

Option 1: Use your actual project screenshots
1. Take screenshots of your projects/applications
2. Resize to recommended dimensions
3. Compress to reduce file size
4. Name files exactly: desktop.png, mobile.png
5. Replace the placeholder files in this folder

Option 2: Create mockups
1. Use design tools (Figma, Sketch, Canva)
2. Create mockups of your projects
3. Export as PNG at recommended sizes
4. Replace placeholder files

Option 3: Temporary placeholders (for testing)
You can use online placeholder services temporarily:
- https://placeholder.com/1920x1080
- https://via.placeholder.com/1200x800.png
- https://placehold.co/1200x800/0a192f/00d9ff?text=Project+Screenshot

COMPRESSION TOOLS:
------------------
- TinyPNG: https://tinypng.com (free, drag & drop)
- ImageOptim: https://imageoptim.com (Mac app)
- Squoosh: https://squoosh.app (browser-based)
- GIMP: https://www.gimp.org (free desktop app)

SCREENSHOT TOOLS:
-----------------
Browser DevTools (built-in):
1. Open DevTools (F12)
2. Press Ctrl+Shift+P (Cmd+Shift+P on Mac)
3. Type "screenshot"
4. Choose "Capture full size screenshot"

Desktop tools:
- Greenshot (Windows): https://getgreenshot.org
- Snagit (Windows/Mac): https://www.techsmith.com/screen-capture.html
- macOS built-in: Cmd+Shift+4

BEST PRACTICES:
---------------
1. Show actual interface of your projects
2. Include some UI elements (buttons, forms, data)
3. Avoid empty/blank screenshots
4. Use high-contrast images
5. Consider dark theme to match portfolio
6. Add annotations/highlights for key features (optional)

EXAMPLE FILENAMES:
------------------
If you have multiple projects, create variations:
- desktop.png (project 1)
- desktop-project2.png
- desktop-project3.png
- mobile.png (mobile project)

Then update image src in index.html to point to correct file.

NOTE:
-----
These placeholder images are generated as simple gray rectangles.
The portfolio will still function with these placeholders, but
your deployed site should use actual project screenshots for
a professional appearance.
